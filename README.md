# 리눅스 명령어 top, ps, jobs, kill 조사하기

---
## top

'top'명령어는 리눅스 시스템에서 실행 중인 프로세스와 시스템의 자원 사용 현황을 실시간으로 보여주는 강력한 유틸리티입니다.

* 기본 사용법:
```
top [option]
```

- 주요 기능:
1. **실시간 모니터링** : 'top' 명령어는 일정 주기로 화면을 업데이트하여 시스템 성능을 실시간으로 모니터링 할 수 있습니다.  
2. **프로세스 정보** : 실행 중인 프로세스의 PID, 사용자, 우선순위, 메모리 사용량, CPU사용량 등을 보여줍니다.  
3. **시스템 요약** : 화면 상단에 시스템의 전반적인 성능 지표를 요약하여 표시합니다. 이는 시스템 가종 시간, 사용자 수, 로드 평균, CPU 상태, 메모리 사용량, 스왑 공간 사용량 등을 포함합니다.  

- 주요 옵션:
| 옵션 | 설명 |
| :-- | :-- |
| -d delay | 지정한 시간(delay) 간격으로 정보를 출력 |
| -n num | 지정한 횟수(num)만큼 정보를 출력 |
| -p pid | 지정한 프로세스 id(pid)의 정보만 출력 |

## ps

- ps 명령어는 현재 시점의 프로세스 상태를 정적으로 보여줍니다.

- 기본 사용법:
  ```
  ps [option]  
  ps aux
  ```

- 주요 옵션:
1. **aux** : 모든 사용자의 모든 프로세스를 자세히 보여줍니다.
   - **a** : 터미널과 관련된 모든 프로세스
   - **u** : 사용자 기반 형식으로 출력
   - **x** : 터미널과 관련되지 않은 프로세스 포함

2. **ef** : 계층적 구조(부모-자식 관계)를 포함하여 자세히 보여줍니다.
   - **e** : 모든 프로세스 표시
   - **f** : 풀 포맷

## jobs

- jobs 명령어는 현재 셀에서 실행 중인 백그라운드 작업 목록을 보여줍니다.
  이 명령어는 주로 작업 제어와 관련된 정보를 제공하며 사용자에게 어떤 작업이 실행 중인지, 멈춰 있는지 등을 확인할 수 있게 해줍니다.

- 주요 기능:
 1. **작업 번호**
 2. **작업 상태**
 3. **작업이 실행된 명령어**

- 기본 사용법:
  ```
  jobs [option][job]
  ```

- 출력되는 상태값:

| 상태 | 설명 |
| :-- | :-- |
| Running | 작업이 계속 진행중임 |
| Done | 작업이 완료되어 0을 변환 |
| Done(code) | 작업이 종료되었으며 0이 아닌 코드를 변환 |
| Stopped | 작업이 일시 중단 |

## kill

- kill 명령어는 특정 프로세스에 시그널을 보내 해당 프로세스를 종료하거나 제어하는 데 사용됩니다.
  
- 구문:
  ```
  kill [시그널][PID]
  ```

- 자주 사용되는 시그널:
1. **1(SIGNUP)**: 프로세스 재시작 또는 설정 파일 다시 읽기 요청(터미널 종료 시 발생)
2. **9(SIGKILL)**: 강제 종료. 프로세스가 시그널을 무시할 수 없는 가장 강력한 종료 시그널입니다. (데이터 손실 가능성이 있음)
3. **15(SIGTERM)**: 정상적인 종료 요청. 프로세스가 종료 전에 정리 작업을 수행할 수 있도록 허용합니다.(가장 권장되는 종료 방식)

- 사용 예시:
```
#프로세스 ID 12345를 정상적으로 종료 요청
kill 15 12345

#프로세스 ID 54321을 강제 종료
kill -9 54321
```
